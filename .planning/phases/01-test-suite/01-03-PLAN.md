**Helper:** Create a function `make_entry(StringTable& st, ...)` that constructs a LogEntry with given severity, component, namespace, msg, duration, conn_id, timestamp. Interns string fields via the StringTable and returns a populated LogEntry.

```cpp
static LogEntry make_entry(StringTable& st,
                           Severity sev, const char* comp, const char* ns,
                           const char* msg, const char* op_type,
                           int32_t duration, uint32_t conn_id,
                           int64_t ts_ms) {
    LogEntry e{};
    e.severity = sev;
    e.component_idx = st.intern(comp);
    e.ns_idx = st.intern(ns);
    e.msg_idx = st.intern(msg);
    e.op_type_idx = st.intern(op_type);
    e.duration_ms = duration;
    e.conn_id = conn_id;
    e.timestamp_ms = ts_ms;
    return e;
}
```

**Test cases using `QueryShapeNormalizer::normalize(string_view)`** (the slow path that parses JSON then normalizes):

1. **"normalize empty string"** — `normalize("")` returns `""`.

2. **"normalize simple string value"** — `normalize("\"hello\"")` returns `"{str}"`.

3. **"normalize integer"** — `normalize("42")` returns `"{int}"`.

4. **"normalize float"** — `normalize("3.14")` returns `"{num}"`.

5. **"normalize boolean"** — `normalize("true")` returns `"{bool}"`. `normalize("false")` returns `"{bool}"`.

6. **"normalize null"** — `normalize("null")` returns `"{null}"`.

7. **"normalize simple object"** — `normalize("{\"name\":\"John\",\"age\":30}")` returns `"{age:{int},name:{str}}"` (keys sorted alphabetically).

8. **"normalize key sorting"** — `normalize("{\"z\":1,\"a\":2,\"m\":3}")` returns `"{a:{int},m:{int},z:{int}}"`.

9. **"normalize nested object"** — `normalize("{\"filter\":{\"email\":\"x\"},\"sort\":{\"date\":1}}")` returns `"{filter:{email:{str}},sort:{date:{int}}}"`.

10. **"normalize array"** — `normalize("[1,\"two\",3.0]")` returns `"[{int},{str},{num}]"`.

11. **"normalize empty object"** — `normalize("{}")` returns `"{}"`.

12. **"normalize empty array"** — `normalize("[]")` returns `"[]"`.

13. **"normalize extended JSON $oid"** — `normalize("{\"_id\":{\"$oid\":\"507f1f77bcf86cd799439011\"}}")` returns `"{_id:{oid}}"`.

14. **"normalize extended JSON $date"** — `normalize("{\"created\":{\"$date\":\"2024-01-01T00:00:00Z\"}}")` returns `"{created:{date}}"`.

15. **"normalize extended JSON $regex"** — `normalize("{\"pattern\":{\"$regex\":\"^test\",\"$options\":\"i\"}}")` returns `"{pattern:{regex}}"`.

16. **"normalize extended JSON $numberLong"** — `normalize("{\"count\":{\"$numberLong\":\"123456\"}}")` returns `"{count:{num}}"`.

17. **"normalize extended JSON $binary"** — `normalize("{\"data\":{\"$binary\":{\"base64\":\"SGVsbG8=\",\"subType\":\"00\"}}}")` returns `"{data:{bin}}"`.

18. **"normalize extended JSON $timestamp"** — `normalize("{\"ts\":{\"$timestamp\":{\"t\":1234,\"i\":1}}}")` returns `"{ts:{ts}}"`.

19. **"normalize MongoDB query shape"** — Test a realistic query:
    ```json
    {"find":"users","filter":{"email":"test@example.com","status":{"$in":["active","pending"]}},"sort":{"created":-1}}
    ```

20. **"normalize invalid JSON fallback"** — `normalize("not json")` returns the original string `"not json"` (error fallback).
