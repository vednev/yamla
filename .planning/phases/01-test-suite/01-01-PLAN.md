   Add these Makefile additions:
   ```makefile
   # ---- Test target -------------------------------------------
   # Catch2 flags from Conan
   CATCH2_CFLAGS := $(shell PKG_CONFIG_PATH=$(CONAN_PC) pkg-config --cflags catch2-with-main 2>/dev/null)
   CATCH2_LIBS   := $(shell PKG_CONFIG_PATH=$(CONAN_PC) pkg-config --libs   catch2-with-main 2>/dev/null)

   # Test source files
   TEST_SRCS := $(shell find test -name '*.cpp' 2>/dev/null)
   TEST_OBJS := $(patsubst test/%.cpp, $(BUILDDIR)/test/%.o, $(TEST_SRCS))

   # Source files needed by tests (exclude main.cpp and ui/ — they need SDL/ImGui)
   TEST_DEP_SRCS := $(filter-out src/main.cpp, $(filter-out src/ui/%, $(filter-out src/llm/llm_client.cpp, $(SRCS))))
   TEST_DEP_OBJS := $(patsubst $(SRCDIR)/%.cpp, $(BUILDDIR)/%.o, $(TEST_DEP_SRCS))

   TEST_CXXFLAGS := $(CXXFLAGS) $(CATCH2_CFLAGS)
   TEST_LDFLAGS  := $(CATCH2_LIBS) $(CONAN_LIBS) $(SSL_LIBS)

   test: $(TEST_OBJS) $(TEST_DEP_OBJS)
   	$(CXX) $(TEST_CXXFLAGS) -o build/run_tests $^ $(TEST_LDFLAGS)
   	./build/run_tests

   $(BUILDDIR)/test/%.o: test/%.cpp
   	@mkdir -p $(dir $@)
   	$(CXX) $(TEST_CXXFLAGS) -c $< -o $@
   ```

   Also add `test` to the `.PHONY` line (change `.PHONY: all deps clean run` to `.PHONY: all deps clean run test`).

   **IMPORTANT:** The TEST_DEP_SRCS filter excludes `src/main.cpp` (has its own `main()`), all of `src/ui/*` (depends on SDL2/ImGui headers), and `src/llm/llm_client.cpp` (depends on httplib.h and OpenSSL). Tests for those modules' pure-logic functions will use separate test-only compilation units that extract the testable functions. For this plan, we only test header-only core modules (arena, chunk_vector) which don't need separate .cpp compilation.


**ArenaAllocator tests:**

1. **"ArenaAllocator basic allocation"** — Create ArenaAllocator(1024). Allocate 64 bytes. Check `used() >= 64`, `remaining() <= 960`, returned pointer is non-null.

2. **"ArenaAllocator alignment"** — Create ArenaAllocator(4096). Allocate 1 byte with alignment=1, then allocate 8 bytes with alignment=8. Check second pointer is 8-byte aligned: `(reinterpret_cast<uintptr_t>(p2) % 8) == 0`.

3. **"ArenaAllocator alloc_zeroed"** — Create ArenaAllocator(1024). Call `alloc_zeroed(128)`. Check all 128 bytes are zero using `memcmp` against a zero-filled array.

4. **"ArenaAllocator alloc_one and alloc_array"** — Create ArenaAllocator(4096). Call `alloc_one<int64_t>()` — check pointer is non-null and 8-byte aligned. Call `alloc_array<int32_t>(10)` — check pointer is non-null, write values 0-9, read them back.

5. **"ArenaAllocator intern_string"** — Create ArenaAllocator(1024). Intern "hello" (len=5). Check returned pointer is non-null, `strcmp(result, "hello") == 0`, and the returned pointer's address is within the arena's slab memory range (different from the source string).

6. **"ArenaAllocator reset"** — Create ArenaAllocator(1024). Allocate 512 bytes. Check `used() >= 512`. Call `reset()`. Check `used() == 0`, `remaining() == capacity()`.

7. **"ArenaAllocator multiple allocations fill up"** — Create ArenaAllocator(256). Allocate 100 bytes three times (300 bytes > 256 capacity due to alignment). This should trigger an assertion failure. Use Catch2's signal-based death test if available, or simply verify `remaining()` decreases correctly for allocations that fit.

**ArenaChain tests:**

8. **"ArenaChain starts with one slab"** — Create ArenaChain. Check `slab_count() == 1`, `total_capacity() == ArenaChain::SLAB_SIZE`.

9. **"ArenaChain basic allocation"** — Create ArenaChain. Allocate 1024 bytes. Check returned pointer is non-null, `approximate_used() >= 1024`.

10. **"ArenaChain intern_string stability"** — Create ArenaChain. Intern "test_string_1". Intern "test_string_2". Check both returned `const char*` are non-null, `strcmp` against originals, and pointers remain valid (not invalidated by second intern).

11. **"ArenaChain reset keeps one slab"** — Create ArenaChain. Allocate some memory. Call `reset()`. Check `slab_count() == 1`, `approximate_used() == 0`.

12. **"ArenaChain alloc_array"** — Create ArenaChain. Allocate `alloc_array<uint64_t>(100)`. Write values 0-99, read them back. All must match.

Note: We skip testing multi-slab growth exhaustively (would require allocating 256MB+) but do include a test that allocates enough to verify the chain grows. Use a smaller allocation pattern: allocate many 1MB blocks until slab_count() > 1.

13. **"ArenaChain grows to multiple slabs"** — Create ArenaChain. Allocate 1MB blocks in a loop. After ~260 allocations (260MB > 256MB slab), check `slab_count() >= 2`. All earlier pointers must still be dereferenceable (write a byte to each, read back).

```cpp
#include <catch2/catch_all.hpp>
#include "core/chunk_vector.hpp"
#include <numeric>
```

Use a small CHUNK_CAPACITY for tests to make chunk boundaries testable without allocating huge memory. Use `ChunkVector<int, 4>` (4 elements per chunk) for most tests, and `ChunkVector<int>` (default 65536) for one realistic test.

1. **"ChunkVector empty on construction"** — Create ArenaChain + ChunkVector<int,4>. Check `size() == 0`, `empty() == true`, `chunk_count() == 0`.

2. **"ChunkVector push_back and random access"** — Push 10 ints (0-9). Check `size() == 10`, `empty() == false`. Check `cv[0] == 0`, `cv[5] == 5`, `cv[9] == 9`.

3. **"ChunkVector chunk boundaries"** — Use ChunkVector<int,4>. Push 8 items. Check `chunk_count() == 2` (4 items per chunk, 8 items = 2 chunks). Push 1 more. Check `chunk_count() == 3`.

4. **"ChunkVector write through operator[]"** — Push 5 ints. Set `cv[2] = 42`. Read back `cv[2]` — must be 42. Other elements unchanged.

5. **"ChunkVector sort ascending"** — Push values {5, 3, 8, 1, 4, 7, 2, 6} into ChunkVector<int,4>. Create scratch ArenaChain. Call `cv.sort(scratch, std::less<int>())`. Verify elements are now {1, 2, 3, 4, 5, 6, 7, 8} via sequential read.

6. **"ChunkVector sort descending"** — Push values {1, 2, 3, 4, 5, 6, 7, 8}. Sort with `std::greater<int>()`. Verify {8, 7, 6, 5, 4, 3, 2, 1}.

7. **"ChunkVector sort single element"** — Push 1 element. Sort. Element unchanged. `size() == 1`.

8. **"ChunkVector sort empty"** — Sort empty vector. No crash. `size() == 0`.

9. **"ChunkVector iterator"** — Push 5 values. Use range-for loop to collect into `std::vector<int>`. Compare against expected.

10. **"ChunkVector const iterator"** — Push 5 values. Create `const auto&` reference. Use range-for on const ref. Verify values match.

11. **"ChunkVector clear"** — Push 10 values. Call `clear()`. Check `size() == 0`, `empty() == true`.

12. **"ChunkVector large dataset sort"** — Use default chunk capacity. Push 100,000 values in reverse order (100000, 99999, ..., 1). Sort ascending. Verify first element is 1, last is 100000, and each `cv[i] <= cv[i+1]` for all i.
